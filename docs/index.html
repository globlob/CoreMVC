Core Framework is right for me?

  • You wanna learn MVC structure
  • You wanna make easy your projects

Install - Setup

Core FW give 3 options to you

With Composer

If you will use composer you only need 2 steps:
  • Open your command-line tool(Terminal) and go to target folder with cd command
  • And write on consol this : composer create-project tolgabulca/coremvc

With Git

If you don't have composer but you have Git you only need 2 steps:
  • Open your command-line tool(Terminal) and go to target folder with cd command
  • And write on consol this : git clone https://github.com/tolgabulca/CoreMVC.git

With Git

If you don't have composer and git you need 4 steps:
  • Open your browser and write this: https://github.com/tolgabulca/CoreMVC
  • You must see Download ZIP button click on it
  • Open the ZIP file on target folder
Core only works on a LAMP folder like xampp/htdocs

General

Controllers

Controllers are the 'back-end' files of your application. You can handle HTTP request's and you can connect model to view.

Root File:

Your root file '/' is always must be 'home.php'

File naming:

Your file name must be same as your class name.

First Try:

Let's create your first app with a controller; create a file and rename to 'hello.php' and let's write some code: 

    class Hello extends Controller {
    public function index(){
    echo 'Hello World';
    }
    }
Now visit your site using this URL:
example.com/hello
If you see Hello World you made it right

Methods

Of course you need more than one method in your controller. The index() method is root method of your controller file. Don't need write method name on adress bar if you are in index() method. But else you need write method name on adress bar like this:
example.com/Hello/Method
Let's create a method: 

    class Hello extends Controller {
    public function index(){
    echo 'Hello World';
    }
    public function say_hi($name = ''){
    echo 'Hi $name';
    }
    }
Okey, you will receive name with 'parameter' you will accept dozens of parameters in your method and how do you use it if you want.

Views

Views are the skin of your application and formed only HTML,JS,CSS and some PHP you don't need write complex code.

Create a View

You just write standart HTML tags with your code editor and save it on app/views folder

Call Views

Call views with this method: $this->view()
An example: 

    class Home extends Controller{
    public function index($name = ''){
    $data = ['text'=> $name];
    $this->view('home/index',$data);
    }
    }

Using Multiple Views

You can use multiple view in Controller file: 

    class Home extends Controller{
    public function index($name = ''){
    $data = ['text'=> $name];
    $this->view('home/header');
    $this->view('home/index',$data);
    $this->view('home/footer');
    }
    }

Send data to Views

A task is controller file separate php codes to html,too. If you need send data to view you must be use $data array like this: 
$data = ['text'=> $name];
    $this->view('home/index',$data);
Only submit data to view if you need
And use the data like this: 
echo $data['text']

Models

Models are brain of your application. They are tasks the the data. This is a model structure: 

    use Illuminate\Database\Eloquent\Model as Eloquent;
    class Home extends Eloquent
    {
    protected $table = 'book';
    protected $fillable = ['id','msg'];

    public function update_entry()
    {
    self::create(array('msg' => 'Hello'));
    }
    public function get_all_msg()
    {
    return self::all();
    }
    }

Anathomy

Models are stored in app/models folder. It is a basic example: 

    use Illuminate\Database\Eloquent\Model as Eloquent;
    class Home extends Eloquent
    {
    protected $table = 'book';
    }
Most of the time Eloquent find the table but sometimes you need add table name.

Call Model

Call models like this; 

    $this->model('home');
If you call the model in __construct method you use it anywhere

File naming:

Your file name must be same as your class name.
Get more information from http://laravel.com/docs/4.2/eloquent

Widgets

Widgets the helpers of your applications. Make easy your works.

Call Widgets

Call widgets with this method: $this->getWidget('widget')
An example: 

    class Home extends Controller{
    public function index(){
    $this->getWidget('form');
    }
    }
If you call the widget in __construct method you use it anywhere

Using Multiple Widgets

You can use multiple view in Controller file: 

    class Home extends Controller{
    public function index(){
    $this->getWidget('form');
    $this->getWidget('cache');
    $this->getWidget('log');
    }
    }

Cache

Caching is sometimes recover you dozens of bytes.

Call Cache

Call cache with this method: $this->getWidget('cache')
An example: 

    class Home extends Controller{
    public function index(){
    $this->getWidget('cache');
    }
    }
Cache files stored in app/cache folder

Usage

Cache widget usage more harder then other widgets but more optional. 

    class Home extends Controller{
    public function __construct(){
    $this->getWidget('cache');
    }
    public function index($name = ''){
    $data = ['text'=> $name];
    $cache = new cache();
        $cache->app_url = '';
        $cache->caching = false;
        $cache->cacheStart();
    $this->view('home/index',$data);
    $cache->cacheEnd();
    }
    }
There are 3 parameters cache_time,app_url and caching;
  • cachetime: This parameter determine how often renewed cache file.
  • app_url: This parameter is name your application folder if you stored application in root leave this parameter empty.
  • caching: Sometimes you don't need caching or stop cache like dynamic page. Use 'caching' parameter if cache parameter equals false stop caching

There are 2 methods cacheStart and cacheEnd;
  • cacheStart: This method start caching
  • cacheEnd: This method end caching
Cached files are saved md5 names you don't find file name

Form

A widget for make easy build HTML forms.

Call Form Widget

Call cache with this method: $this->getWidget('form')
An example: 

    class Home extends Controller{
    public function index(){
    $this->getWidget('form');
    }
    }

Usage

Form widget use in view files. Like this: 

    echo openForm('POST','#');
    echo input('text','username','','pleaceholder="Username"');
    echo input('password','password','','pleaceholder="Password"');
    echo input('submit','','Sent');
    echo closeForm();
There are 5 methods openForm(), input(),textarea(),openFormMultipart and closeForm()

  • openForm: This method start the form
    • method: This parameter is a option for select sending HTTP REQUEST
    • action: This parameter is a option for select sending data to which file
    • att: Other attributes
  • input: This method add a form input
    • type: This parameter is a option for select input type
    • name: This parameter is a option for select input name
    • value: This parameter is a option for select input value
    • att: Other attributes
  • textarea: This method add a textarea
    • name: This parameter is a option for select textarea name
    • value: This parameter is a option for select textarea value
    • att: Other attributes
  • openFormMultipart: This method start a multipart form
    • method: This parameter is a option for select sending HTTP REQUEST
    • action: This parameter is a option for select sending data to which file
    • att: Other attributes
  • closeForm: This method close the form

Call Log Widget
